Lifecycle Hooks

Metodos que Polymer ejecuta en momentos clave: creacion, montaje y desmontaje del componente.

Donde se usa

En cualquier componente donde necesites ejecutar codigo en un momento concreto: al crearse, al insertarse en el DOM o al quitarse.

Como se usa

Sobreescribiendo los metodos y llamando siempre super.metodo() primero. ready() para setup, connectedCallback() para timers, disconnectedCallback() para limpiarlos.

Que hace

ready() garantiza que el Shadow DOM y las propiedades estan 100% listos. disconnectedCallback() evita memory leaks al limpiar timers y listeners.

vs HTML5 puro

HTMLElement no tiene ready(). El connectedCallback nativo puede ejecutarse antes de que las propiedades esten inicializadas.

HTML5 puro
class MiElem extends HTMLElement {
  connectedCallback() {
    // props pueden NO estar listas
    this.render(); // DOM manual
    this._timer = setInterval(() => {
      this.render(); // re-render manual
    }, 1000);
  }
  disconnectedCallback() {
    clearInterval(this._timer);
  }
}
Polymer
class MiElem extends PolymerElement {
  ready() {
    super.ready();
    // Shadow DOM + props 100% listos
  }
  connectedCallback() {
    super.connectedCallback();
    this._timer = setInterval(() => {
      this.contador++; // UI se actualiza sola
    }, 1000);
  }
  disconnectedCallback() {
    super.disconnectedCallback();
    clearInterval(this._timer);
  }
}
