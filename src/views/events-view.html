Custom Events

Comunicacion entre componentes usando eventos que cruzan el Shadow DOM.

Donde se usa

Cuando un componente hijo necesita comunicar algo al padre: un boton presionado, un valor cambiado, una accion completada.

Como se usa

El hijo llama dispatchEvent(new CustomEvent(...)) con composed: true. El padre escucha con on-nombre="_handler" directo en el template.

Que hace

composed: true permite que el evento salga del Shadow DOM del hijo y suba por el arbol hasta el padre. Sin esto el evento queda atrapado adentro.

vs HTML5 puro

En vanilla los eventos no cruzan el Shadow DOM. Hay que obtener una referencia directa al hijo con querySelector y agregarle un listener a mano.

HTML5 puro
<!-- HTML -->
<mi-hijo id="hijo"></mi-hijo>

// JS hijo — dispara
this.dispatchEvent(new CustomEvent('mi-evento'));
// sin composed:true → queda atrapado

// JS padre — referencia directa obligatoria
const hijo = this.shadowRoot
  .querySelector('#hijo');
hijo.addEventListener('mi-evento', (e) => {
  console.log('recibido', e);
});
Polymer
<!-- HTML hijo -->
<button on-click="_disparar">Click</button>

<!-- HTML padre -->
<mi-hijo on-hijo-click="_recibir"></mi-hijo>

// JS hijo — dispara el evento
_disparar() {
  this.dispatchEvent(new CustomEvent('hijo-click', {
    bubbles: true, composed: true
  }));
}

// JS padre — recibe el evento
_recibir(e) {
  console.log('evento recibido', e.detail);
}
