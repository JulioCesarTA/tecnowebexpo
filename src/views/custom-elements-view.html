Custom Elements

Define tus propias etiquetas HTML reutilizables con comportamiento y estilos propios.

Donde se usa

Cuando queres crear una etiqueta HTML propia que encapsule logica, estilos y template reutilizable en cualquier parte de la app.

Como se usa

Extendiendo PolymerElement, definiendo el template y llamando customElements.define('mi-elem', MiClase). Despues usas <mi-elem> como cualquier etiqueta HTML.

Que hace

Registra un nuevo elemento en el navegador con su propio Shadow DOM, estilos aislados y propiedades reactivas. Cualquier instancia de la etiqueta comparte el mismo comportamiento.

vs HTML5 puro

En vanilla tambien existe customElements.define pero tenes que gestionar manualmente el Shadow DOM, el template y el ciclo de vida. Polymer lo hace declarativo.

HTML5 puro
<!-- HTML -->
<mi-tarjeta nombre="Ana"></mi-tarjeta>

// JavaScript
class MiTarjeta extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }
  static get observedAttributes() {
    return ['nombre']; // declarar atributos a observar
  }
  attributeChangedCallback(attr, anterior, nuevo) {
    // actualizar DOM manualmente al cambiar
    this.shadowRoot.innerHTML = `
      <style>p { color: #1a237e; }</style>
      <p>Hola, ${nuevo}</p>
    `;
  }
}
customElements.define('mi-tarjeta', MiTarjeta);
Polymer
<!-- HTML -->
<mi-tarjeta nombre="Ana"></mi-tarjeta>

// JavaScript
class MiTarjeta extends PolymerElement {
  static get template() {
    // template declarativo con binding
    return html`
      <style>p { color: #1a237e; }</style>
      <p>Hola, [[nombre]]</p>
    `;
  }
  static get properties() {
    return {
      nombre: { type: String, value: '' }
    };
  }
}
customElements.define('mi-tarjeta', MiTarjeta);
