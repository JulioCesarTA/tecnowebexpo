Lifecycle Hooks

Metodos que Polymer ejecuta en momentos clave: creacion, montaje y desmontaje del componente.

Donde se usa

En cualquier componente donde necesites ejecutar codigo en un momento concreto: al crearse, al insertarse en el DOM o al quitarse.

Como se usa

Sobreescribiendo los metodos y llamando siempre super.metodo() primero. ready() para setup, connectedCallback() para timers, disconnectedCallback() para limpiarlos.

Que hace

ready() garantiza que el Shadow DOM y las propiedades estan 100% listos. disconnectedCallback() evita memory leaks al limpiar timers y listeners automaticamente.

vs HTML5 puro

HTMLElement no tiene ready(). El connectedCallback nativo puede ejecutarse antes de que las propiedades esten inicializadas, obligando a checks manuales.

HTML5 puro
<!-- HTML -->
<mi-elemento></mi-elemento>

// JavaScript
class MiElem extends HTMLElement {
  constructor() {
    super();
    // props NO disponibles aun
    this.attachShadow({ mode: 'open' });
  }
  connectedCallback() {
    // puede ejecutarse antes de que
    // los atributos esten inicializados
    this.render(); // DOM manual
    this._timer = setInterval(() => {
      this.render(); // re-render manual
    }, 1000);
  }
  disconnectedCallback() {
    clearInterval(this._timer); // limpiar manual
  }
  render() {
    // actualizar DOM a mano siempre
    this.shadowRoot.innerHTML = `...`;
  }
}
Polymer
<!-- HTML -->
<mi-elemento></mi-elemento>

// JavaScript
class MiElem extends PolymerElement {
  ready() {
    super.ready();
    // Shadow DOM + props 100% listos
    // seguro para acceder al DOM
  }
  connectedCallback() {
    super.connectedCallback();
    // elemento insertado en el DOM
    this._timer = setInterval(() => {
      this.contador++; // Polymer actualiza UI solo
    }, 1000);
  }
  disconnectedCallback() {
    super.disconnectedCallback();
    // elemento removido del DOM
    clearInterval(this._timer);
  }
}
